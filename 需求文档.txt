学生综测成绩系统需求规格说明
1. 系统概述
本系统用于管理学生综合测评成绩，支持学生信息维护、活动信息管理、学生参与活动记录管理、成绩计算、Excel导入导出等功能。系统运行于Windows 11平台，采用Web应用架构（用户要求为Web应用，见第5题选B）。
2. 功能需求
2.1 学生成绩管理
增删改查：对学生基本信息（学号、姓名、院系、年级、班级、学业水平成绩、备注）进行增加、删除、修改、查询。
批量操作：支持导入/导出学生数据（含活动参与记录）。
成绩计算：自动计算素质总分、综合分值，并支持手动调整参与活动的实际得分。
2.2 活动管理
活动表维护：对活动信息（活动编号、活动名称、所属类别、活动分值、是否为进阶）进行增删改查。
所属类别：固定为“身心素养”“文艺素养”“劳动素养”“创新素养”四类。
2.3 学生活动参与记录管理
记录维护：为学生添加、修改、删除参与的活动记录（学号、活动编号、实际得分）。
同一活动允许多次参与：同一学生可多次参加同一活动，每次参与独立记录。
实际得分可调：记录中的“实际得分”字段允许与活动表标准分值不同，若为空则默认使用活动表分值。
关联计算：各素养总分 = 该生在该素养下所有参与记录的实际得分之和（实际得分若为空则取活动表分值）。
2.4 成绩汇总与查询
列表展示：展示学生汇总信息，包括学号、姓名、院系、年级、班级、学业水平成绩、身心素养总分、文艺素养总分、劳动素养总分、创新素养总分、素质总分、综合分值、备注。
排序与筛选：支持按年级、班级、学号、各素养总分、综合分值等进行排序；支持按年级、班级、学号范围、各素养总分范围、综合分值范围筛选。
详情查看：点击某学生可查看该生所有素养下的详细活动参与记录（活动名称、实际得分、活动分值、是否为进阶等）。
2.5 导入导出功能
导入：从用户指定路径的Excel文件（.xlsx）导入数据。支持学生基本信息、学业水平成绩、活动参与记录（以多行文本格式内嵌在素养单元格中）的导入。导入前预览冲突数据，用户可批量选择“覆盖”或“跳过”。
导出：将当前筛选条件下的学生汇总信息及详细活动参与记录导出为Excel文件（可多sheet）。
3. 数据库设计
3.1 表结构
学生表 (students)
字段名	类型	说明
student_id	VARCHAR(20)	学号，主键
name	VARCHAR(50)	姓名
department	VARCHAR(100)	院系
grade	VARCHAR(20)	年级（如“2023级”）
class	VARCHAR(50)	班级
academic_score	DECIMAL(5,2)	学业水平成绩，默认0
remark	TEXT	备注
quality_total	DECIMAL(5,2)	素质总分，由系统自动计算更新
comprehensive_score	DECIMAL(5,2)	综合分值，由系统自动计算更新
活动表 (activities)
字段名	类型	说明
activity_id	VARCHAR(20)	活动编号，主键
name	VARCHAR(200)	活动名称
category	ENUM('身心素养','文艺素养','劳动素养','创新素养')	所属类别
score	DECIMAL(5,2)	标准分值
is_advanced	BOOLEAN	是否为进阶（仅标记，不影响计算）
学生活动参与表 (student_activities)
字段名	类型	说明
id	INT	自增主键
student_id	VARCHAR(20)	学号，外键关联students(student_id)
activity_id	VARCHAR(20)	活动编号，外键关联activities(activity_id)
actual_score	DECIMAL(5,2)	实际得分，为空时取活动表score
participate_date	DATE	参与时间（可选，默认当前日期）
3.2 计算逻辑
素质总分 = SUM(该生所有参与记录的实际得分或活动表分值)
综合分值 = 学业水平成绩 × 0.9 + 素质总分 × 0.1
四个素养总分分别按category汇总计算，不单独存储，查询时实时计算或使用缓存。
4. 导入导出规则
4.1 导入文件格式
使用一个Excel sheet，结构如下（与“学生综测评分表示例.xlsx”类似，但修改活动列格式）：
列名	说明
学号	
院系	
年级	
姓名	
班级	
学业水平成绩	
身心素养活动	单元格内用换行符（\n）分隔多个“活动名称（分值）”条目，如“志愿者活动（2.5）\n心理讲座（1.0）”
文艺素养活动	同上格式
劳动素养活动	同上格式
创新素养活动	同上格式
素质总分	导入时可忽略，系统自动计算
综合分值	导入时可忽略，系统自动计算
备注	
说明：
导入时，系统解析每个素养活动单元格，提取活动名称和分值（括号内数字）。若活动表中不存在该名称，则自动创建活动（活动编号自动生成，所属类别根据列标题确定，分值取括号内数字，is_advanced默认false）。
若活动已存在，则使用已有活动编号建立学生参与记录。
同一学生在同一素养下的多条活动记录，将生成多条student_activities记录。
学生基本信息（学号、姓名、院系、年级、班级、学业水平成绩、备注）按学号更新或插入。
4.2 导入冲突处理
学生信息冲突：以学号为唯一标识。导入时，系统对比现有学生，生成冲突列表，展示新旧数据对比，管理员可勾选每条冲突记录选择“覆盖”（用新数据覆盖原学生信息及活动参与记录）或“跳过”（保留原数据，不处理该学生）。若选择覆盖，原有活动参与记录将被删除并替换为新导入的记录。
活动参与记录冲突：由于覆盖策略中已删除原参与记录，不存在单独的活动记录冲突。
4.3 导出功能
筛选条件：支持按年级、班级、学号范围、各素养总分范围、综合分值范围筛选。
导出内容：包含两个sheet：
Sheet1“学生成绩汇总”：字段同学生成绩列表（含备注）。
Sheet2“活动参与明细”：字段包括学号、姓名、活动名称、所属类别、标准分值、实际得分、是否为进阶、参与时间。
导出格式：.xlsx文件，文件名包含导出时间戳。
5. 界面设计
5.1 整体布局
采用Web页面，顶部导航栏包含“学生管理”“活动管理”“导入/导出”等入口。主区域为数据展示和操作区。
5.2 学生成绩列表页
默认显示所有学生，表格列（按顺序）：学号、姓名、院系、年级、班级、学业水平成绩、身心素养总分、文艺素养总分、劳动素养总分、创新素养总分、素质总分、综合分值、备注。
每行末尾有“详情”“编辑”“删除”按钮。
支持排序：点击列标题可排序（默认按学号升序）。
支持筛选：顶部提供筛选条件输入框（年级、班级、学号范围、各素养总分范围、综合分值范围），点击“搜索”刷新列表。
支持批量导入导出按钮。
备注列支持显示多行文本，鼠标悬停或点击展开。
5.3 学生详情页
展示学生基本信息（可编辑）。
四个素养标签页，每个标签页显示该素养下的活动参与记录列表（活动名称、标准分值、实际得分、参与时间、操作按钮）。
支持添加、修改、删除参与记录。修改实际得分时需校验非负。
保存后自动重新计算素质总分和综合分值并更新学生表。
5.4 活动管理页
独立页面，活动列表展示活动编号、名称、类别、标准分值、是否为进阶。
支持增删改查。删除活动时，若存在学生参与记录，则提示并阻止删除（或级联删除参与记录，由管理员选择）。
5.5 导入导出页面
导入：提供文件选择器，选择Excel文件后，系统解析并展示冲突预览（如有），用户可勾选每条冲突的覆盖/跳过操作，确认后执行导入。
导出：提供筛选条件表单，用户设定后点击“导出”按钮下载文件。
6. 错误处理机制
界面提示：所有操作（增删改查、导入导出、计算等）如发生错误（如数据验证失败、数据库操作异常、文件格式错误等），弹出模态框显示具体错误信息。
日志记录：系统自动记录错误日志到文件（如log.txt），包含时间、错误类型、堆栈信息，便于排查。
导入异常：如文件格式不符合要求，提示用户并中止导入；如解析部分数据出错，记录错误行到日志，并提示用户检查后重试。
7. 技术选型建议（供开发参考）
后端：Python Django 
前端：Vue.js + Element UI 
数据库：SQLite（轻量级，便于部署）或 MySQL（性能更强）。
Excel处理：openpyxl
运行环境：Windows 11，Python 3.8+，浏览器建议使用Chrome/Edge。
